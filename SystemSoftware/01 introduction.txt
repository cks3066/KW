컴퓨터 시스템 구조
	하드웨어 OS 어플리케이션 유저

OS란 무엇인가?
	1.유저 프로그램들을 실행
	2.개발자가 어떤 문제를 쉽게 풀수 있도록 기본 서비스를 제공하는 라이브러리(시스템 콜)를 제공
	3.컴퓨터 하드웨어(CPU,DRAM 등)을 효율적으로 사용
	정의 : 사용자와 컴퓨터 하드웨어 사이에 존재하는 아주 큰 덩어리의 프로그램
	- CPU, 메모리, I/O 장치들을 컨트롤, 할당하는 시스템 소프트웨어이다.
시스템 구조
	쉘		워드	컴파일러
O |	시스템 콜 인터페이스(함수)   			 <- 우리가 배울 부분
S. | [커널]	프로세스	메모리	파일	네트워크   [관리]     	
		I/O 메니지먼트
	CPU,캐시,메모리	Disk	모니터	네트워크 카드

시스템 소프트웨어 (or 시스템 프로그램)
	시스템 소프트웨어란 개발자가 프로그램을 개발을 하거나 그리고 프로그램을 실행할 때 필요로하는 환경
	-개발 시 필요로 하는 툴
	> 파일 매니지먼트
	> 상태 정보
	> 프로그래밍 언어 지원
	> 프로그램 로딩 및 실행
	> 커뮤니케이션

	파일 관리(파일 매니지먼트)
		cp, rm , mkdir, ls
		copy / remove / make directory / list
	상태 정보 (스테터스 인포메이션)
		-date, time, 사용 가능 메모리, 디스크 공간, 유저 넘버 등등
		-해킹당하면 누가 사용했는지 이벤트 로그를 보고 누가 썼는지 알 수 있다.
	프로그래밍 언어 지원
		컴파일러, 어셈블러, 디버거, 인터프리터
		-gcc, VIsual studio, javac
	프로그램 로딩, 실행
		엔트리 포인터 <- 시작함수 (메인함수)
		메인 함수를 시작하는 것이 로더(loader) 이것이 시스템 소프트웨어 [ ld ]
		링커[ linkage editor ]
	커뮤니케이션
		telnet, ping, kill
시스템 프로그래밍이란 무엇인가?
	시스템 콜 또는 어셈블리 언어로 시스템 프로그램을 개발하는 것
	(그동안 C나 C++을 통해서 프로그래밍 > 어플리케이션 프로그래밍)
	시스템 콜 또는 어셈블리 언어로 프로그래밍하는 것을 시스템 프로그램이라고 한다.
시스템 콜 & API [함수]
	시스템 콜
		OS에 의해 제공되는 프로그래밍 인터페이스(함수)
	대부분의 OS는 C로 만들어져 있다. (linux, window, mac OS 전부 C로 만들어져 있다.)
	API (어플리케이션 프로그래밍 인터페이스)
		앱 개발자에게 제공되는 함수
		시스템 콜을 이용해서 앱 개발시 훨씬 어려움
		API가 시스템 콜보다 사용하기 훨씬 편리하다.
		ex) printf(), getch(), malloc(), fopen(), fclose()


  App
________  ^^^  	 어플리케이션 프로그래밍 인터페이스(API) / fopen, fread, printf()
  미들 웨어 (middle ware)   // 라이브러리(스탠다드 C) / 자바 버츄어 머신 (JVM) -> API 제공
________  ^^^  	 시스템 콜 인터페이스 /open(), read(), write() 
   OS


개발자가 open() 이라는 시스템콜을 호출하면 (시스템 콜 인터페이스가 지원해줌) OS 안으로 들어감
OS안에 있는 코드들이 호출이 되서 실행됨 그 후에 유저 어플리케이션에 실행됨(리턴)
[시스템 콜을 실행하면 무조건 OS로 들어간 뒤 사용자 어플리케이션에 실행됨]
개발자는 절대 OS를 건드리거나 만질수 없도록 되어있음 > 시스템콜을 통해서 안에 있는 코드를 실행할 뿐

우리가 짜는 프로그램들은 유저모드에서만 실행되는 프로그램들임

예시 : 스탠다드 C 라이브러리 [미들웨어]
printf("안녕");
> 스탠다드 C 라이브러리
> OS 안으로 들어가서 write() 함수 호출 (코드 실행) 
> 유저 화면으로 돌아와서 "안녕"이 실행됨
즉 C언어상 printf라는 코드는 OS에 있는 write()에 대한 함수를 불러오고 실행하겠다는 허용을 얻어오는 함수
